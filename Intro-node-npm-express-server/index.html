What is Node.js?

  • Open-source, cross-platform runtime environment
  • Built on Chrome’s V8 JavaScript engine
  • Used for server-side applications
  • Single-threaded, event-driven architecture
Node.js is free, works on all OS, and uses Chrome’s V8 engine. It allows JavaScript outside the browser, mainly for backend apps. Its single-threaded, event-driven design makes it lightweight but scalable.

Why Node.js?

  • Fast and scalable
  • JavaScript everywhere (frontend + backend)
  • Rich ecosystem of npm packages
  • Non-blocking I/O model
Node.js is fast because of V8 + async I/O. Developers use one language across frontend & backend. npm offers millions of packages. Its non-blocking model means tasks don’t block each other.

Node.js Installation

# Check Node.js version
node -v

# Check npm version
npm -v
Make sure Node is installed from nodejs.org. Use these commands to confirm installation.

npm (Node Package Manager)

  • Default package manager for Node.js
  • Over 2M+ packages available
  • Helps manage dependencies
# Initialize project
npm init -y

# Install a package
npm install express
npm helps install and manage libraries. `init` creates package.json, `install` downloads modules.

package.json

{
  "name": "my-app",
  "version": "1.0.0",
  "scripts": {
    "start": "node index.js"
  },
  "dependencies": {
    "express": "^4.18.2"
  }
}
package.json stores metadata, scripts, and dependencies. Think of it as a project manifest.

What is Express.js?

  • Fast, minimal web framework for Node.js
  • Simplifies server creation
  • Middleware support
  • Routing made easy
Express is lightweight, easy to use, and adds routing + middleware to Node.js.

Create Express.js Server

const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000');
});
`require` imports Express. `app.get` defines routes. `listen` runs the server.

Express.js Routing

app.get('/about', (req, res) => {
  res.send('About Page');
});

app.post('/login', (req, res) => {
  res.send('Login Success');
});
Routes define how the app responds to URLs. GET is for reading, POST is for sending data.

Middleware in Express

app.use((req, res, next) => {
  console.log('Request URL:', req.url);
  next();
});

Middleware functions can modify requests/responses.

Middleware sits between request & response. Use it for logging, authentication, or error handling.

Sync vs Async in Node.js

Synchronous Code

const fs = require("fs");

console.log("Start");

const data1 = fs.readFileSync("file1.txt", "utf8"); // blocking
console.log("File1 content:", data1);

const data2 = fs.readFileSync("file2.txt", "utf8"); // blocking
console.log("File2 content:", data2);

console.log("End");

Asynchronous Code

const fs = require("fs");

console.log("Start");

fs.readFile("file1.txt", "utf8", (err, data1) => {
  if (err) throw err;
  console.log("File1 content:", data1);
});

fs.readFile("file2.txt", "utf8", (err, data2) => {
  if (err) throw err;
  console.log("File2 content:", data2);
});

console.log("End");
• Sync: Execution is Start → File1 → File2 → End (blocking).
• Async: Execution is Start → End → File1 → File2 (non-blocking).
• This shows Node.js’s non-blocking, event-driven model.

Event Loop Analogy

Imagine a smarter restaurant:

  • There is one main waiter (event loop).
  • When a customer orders food, the waiter passes the order to the kitchen (OS/worker threads).
  • The waiter immediately goes to take the next customer’s order.
  • When the kitchen finishes cooking, they notify the waiter, who delivers it to the right customer.
This analogy shows why Node.js can handle thousands of requests with a single thread. The waiter (event loop) never stands idle; the kitchen (system) does the heavy work.
